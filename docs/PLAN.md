# AI6130 Group Project Plan

## Group G30 | Detection of AI-Generated Social Media Text

---

## 1. Project Objective

### The Problem

AI models like ChatGPT, Claude, and Gemini can now write text virtually indistinguishable from human writing. AI-generated content is flooding social media platforms (Twitter, Reddit, Telegram, etc.), raising concerns about fake news, opinion manipulation, and spam.

### What We Will Build

A system that automatically classifies whether a social media post was **written by a human or generated by AI**.

When a user pastes a post into the web interface, the system returns:

1. **Verdict**: "AI-generated" or "Human-written"
2. **Confidence score**: e.g., "87% confident"
3. **Evidence**: "5 similar texts found, 4 were AI-generated"
4. **(Stretch goal)** Source attribution: e.g., "Likely generated by GPT-4"

### Core Research Question

> "Does combining fine-tuning with retrieval-augmented classification improve generalization across different AI models and social media platforms?"

---

## 2. How the Detection Works

Two complementary approaches, each covering the other's weakness.

### Approach 1: Find Similar Texts and Compare (Retrieval)

Pre-build a database of 500,000+ labeled texts. When a new text arrives, find the 10 most similar texts. If 7/10 are labeled "AI," the new text is likely AI-generated.

We convert texts to numerical vectors ("embeddings") using **Gemini Embedding 2** (Google). Similar texts produce similar vectors. **FAISS** (Meta) rapidly finds the closest vectors in the database.

- **Strength**: New AI models handled by adding outputs to database — no retraining
- **Weakness**: Unreliable for very short or ambiguous texts

### Approach 2: Let AI Judge (Fine-tuning)

Take **Qwen 2.5** (32B parameters) and teach it AI vs. human classification via **QLoRA** — freezing 99% of the model and training only ~1%.

- **Strength**: Learns precise patterns from training data
- **Weakness**: Struggles with AI models not seen during training

### Why Combine Both?

Retrieved similar texts are provided as reference context to the fine-tuned LLM. The model provides strong reasoning, retrieved examples provide evidence, and new AI models can be incorporated by updating the database alone.

## 3. Data Preparation

### 3.1 Datasets

#### MultiSocial (Primary) — ACL 2025, 472K texts, 7 AI models, 5 platforms, 22 languages. Free from Zenodo.

#### RAID (Adversarial Eval Only) — ACL 2024, 10M+ texts, 11 AI models, 12 attack types. Testing robustness only.

#### HC3 (Supplementary) — 37K QA pairs (human vs. ChatGPT). Early prototyping.

#### Self-Generated (Optional) — ~50,000 texts (5 frontier models × 10,000). Cost: ~$7–12. Decision after initial experiments.

### 3.2 Preprocessing

Format unification → text normalization → label alignment across all datasets.

### 3.3 Data Splits

Training (70%) / Validation (15%) / Test (15% + five special test scenarios).

### 3.4 Embedding and FAISS Index Construction

1. Convert corpus to 768-dim vectors via Gemini Embedding 2 API
2. Store in FAISS index (~3GB RAM)
3. For each training sample, retrieve 10 nearest neighbors (excluding self)
4. Bake neighbors into training prompts (matches inference format)

### 3.5 Embedding Model Selection

| Attribute | Gemini Embedding 2 (Priority 1) | OpenAI text-embedding-3-large (Priority 2) | BGE-M3 (Priority 3) |
|-----------|------|------|------|
| **MTEB Score** | 68.32 (#1) | 64.6 | 63.0 |
| **Classification boost** | task_type=CLASSIFICATION (+7.1) | None | None |
| **Cost / 1M tokens** | $0.20 | $0.13 | Free (local) |
| **API stability** | Preview (uncertain) | Stable (2+ years) | N/A (local) |
| **Internet required** | Yes | Yes | No |

Code should abstract the embedding function for easy switching. Switching models requires re-embedding the entire corpus.

### 3.6 Data Sizing Rationale

| Purpose | Size | Rationale |
|---------|------|-----------|
| QLoRA training | 50K–100K | Saturation begins here |
| FAISS index | 300K–500K | Sufficient diversity/density |
| Each test set | 5K–10K | 95% CI within ±1–1.4% |
| Self-generated | 10K/model, 50K total | Sufficient gap coverage |

Bonus: Data scaling experiment (10K/25K/50K/100K) to plot the saturation curve.

## 4. Experiment Design

### 4.1 Six Model Configurations

**Ablation Study:**

| # | Name | Description | Purpose |
|---|------|-------------|---------|
| 1 | Qwen 32B zero-shot | No training, no retrieval | Baseline floor |
| 2 | Qwen 32B + QLoRA | Fine-tuned, no RAG | Isolate fine-tuning effect |
| 3 | KNN only | Embedding + FAISS vote, no LLM | Isolate retrieval effect |
| 4 | Qwen 32B + QLoRA + RAG | Full pipeline with ensemble | Must be highest to validate thesis |

**External Baselines:**

| # | Name | Description | Purpose |
|---|------|-------------|---------|
| 5 | RoBERTa (pretrained) | GPT-2-era detector, as-is | Most cited baseline |
| 6 | RoBERTa (fine-tuned) | Same model, trained on our data | Fair same-data comparison |

### 4.2 Five Test Scenarios

Standard / Cross-model / Cross-platform / Adversarial / Short text (<100 words).

### 4.3 Metrics

TPR@1%FPR (primary, RAID standard) / AUROC / Macro-F1.
